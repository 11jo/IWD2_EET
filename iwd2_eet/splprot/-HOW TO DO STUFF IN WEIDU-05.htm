    semiticgod said:

    @subtledoctor: That sounds really cool. How does it work?

    What, the toggle? Right now I only have it in concept form. My test install is on an iPad which makes testing kind of slow. But what I've almost fully implemented is, the item sits in your quickslot and uses opcode 146 to cast SpellA; SpellA has timing mode 1 (permanent until death), and raises your caster level by some value (say, 50), and triggers a 177/232 contingency .EFF with condition of hp<102%, triggering SpellB, which cancels itself and has some arbitrary effect that lasts 7 seconds; this will repeat every 6 seconds. There is your arbitrary "Turn Undead" or whatever ability.

    Meanwhile, SpellA will have a 2nd ability header with min_level = 51; this ability will 321 cancel SpellB, 206 protect from SpellB for 7 seconds, and 321 cancel SpellA, restoring your caster level to its appropriate value. So you can (theoretically) activate the aura/ability/whatever, and then cancel it, by just repeatedly using the item. (The item must have the "max charges" in its ability header set to 0, for infinite at-will use.)

    Now, it occurs to me that you can't really do this with spellcasters, because they will suddenly have maximum caster level while the aura/ability is in effect. So you must either a) disable spellcasting during that effect; or b) stop using caster level and convert the conditional effect of SpellA to filter by a stat or spellstate, instead of caster level. The latter is perfectly feasible, on paper at least. I just haven't fully put together the concept. (I'm using this for now with my spontaneous specialist casting mod, which already uses caster level changes anyway, so there is no need (yet) for me to shift it over to a spellstate condition.)

  • argent77 Member Posts: 2,273
    February 7
    The relevant WeiDU functions for dealing with PVRZ are UPDATE_PVRZ_INDICES and INSTALL_PVRZ. Both functions are described in the WeiDU docs.

    Example for installing a BAM V2 file:
    Spoiler
    // 1. Install your BAM V2 file COPY ~%MOD_FOLDER%/bam/myfile.bam~ ~override~ // Called in preparation for step 2 LPF UPDATE_PVRZ_INDICES RET original_base_index new_base_index END // 2. Install associated PVRZ file ACTION_IF (original_base_index >= 0 AND new_base_index >= 0) BEGIN // Only needed if source pvrz page index < 1000. // Alternatively, replace %zero% with a fixed string of zeros in the "source_file" path below. OUTER_PATCH_SAVE zero ~0000~ BEGIN DELETE_BYTES 0 STRING_LENGTH ~%original_base_index%~ END // Install pvrz with updated indices LAF INSTALL_PVRZ INT_VAR original_base_index new_base_index STR_VAR source_file = EVAL ~%MOD_FOLDER%/bam/mos%zero%%original_base_index%.pvrz~ END END
    You would have to expand the second part a bit if your BAM consists of multiple PVRZ files.

  • argent77 Member Posts: 2,273
    February 7 edited February 7
    Some minor corrections. The source_file paths have to be updated, and the "zero" code is not needed in your example.
    Spoiler
    //Baervan Holy Symbol - code gently given by Argent77 COPY ~Deities&Pantheons/Itm/HolySymbols/BAM/BAERS.bam~ ~override~ LPF UPDATE_PVRZ_INDICES RET original_base_index new_base_index END ACTION_IF (original_base_index >= 0 AND new_base_index >= 0) BEGIN LAF INSTALL_PVRZ INT_VAR original_base_index new_base_index STR_VAR source_file = ~Deities&Pantheons/Itm/HolySymbols/BAM/MOS1000.pvrz~ END END COPY ~Deities&Pantheons/Itm/HolySymbols/BAM/BAERL.bam~ ~override~ LPF UPDATE_PVRZ_INDICES RET original_base_index new_base_index END ACTION_IF (original_base_index >= 0 AND new_base_index >= 0) BEGIN LAF INSTALL_PVRZ INT_VAR original_base_index new_base_index STR_VAR source_file = ~Deities&Pantheons/Itm/HolySymbols/BAM/MOS1001.pvrz~ END END

    I'm assuming that PVRZ files are found in the same folder as the BAM files.

    Edit: You can probably place your code into a FOR_EACH loop to remove duplicate code.
    Spoiler
    ACTION_FOR_EACH file IN ~BAERS~ ~BAERL~ BEGIN COPY ~Deities&Pantheons/Itm/HolySymbols/BAM/%file%.bam~ ~override~ LPF UPDATE_PVRZ_INDICES RET original_base_index new_base_index END ACTION_IF (original_base_index >= 0 AND new_base_index >= 0) BEGIN LAF INSTALL_PVRZ INT_VAR original_base_index new_base_index STR_VAR source_file = EVAL ~Deities&Pantheons/Itm/HolySymbols/BAM/MOS%original_base_index%.pvrz~ END END END
